Using PostgreSQL

In order to use PostgreSQL you must download and install it. The download link is listed above. Just run the installer. During the installation process, it sets the default username to postgres. You will need to supply a password. Write this password down and don't loose it. It will also want to assign the default port of 5432 but you can change this if you wish.

Create a Database

Now that we have PostgreSQL installed we can open pgAdmin. This is what we will be using to build and manage our PostgreSQL databases. The first thing pgAdmin will do is ask for the password you set up when PostgreSQL was installed. Once you enter your password, we are ready to begin.

We are going to create a database that will contain capital names and flags for the world's countries. We have been supplied with two "csv" files that contain the country data and the flag data. Once we have our database built, we can import this data into it.

The first thing we need to do is create a new database.

  1. In pgAdmin, expand the "Servers" list in the Object Explorer (the left hand pane).
  2. Next, right click on "Databases" and select Create, Database
  3. A window will open and on the "General" tab, under Database, enter a name for your database.
  4. Click "SAVE" and your new database has been created. It will now show up in the "Database" list.

Build a Table

We need to set this up in pgAdmin, so in the "Object Explorer", click the arrow next to the world database we just created to expand it, then do the same under the "Schema" element. In the list you will find a section for our Tables.

Right click on Tables and select Query Tool. In the Query window, enter the following code:

  • CREATE TABLE capitals (
    • id SERIAL PRIMARY KEY,
      • SERIAL: will auto-increment the PRIMARY KEY ID number
    • country VARCHAR(45),
    • capitol VARCHAR(45)
      • VARCHAR(45): sets a maximun length for the field to 45 characters
  • )

Press the RUN key from the Query window and your table will be created. Right click on our Tables section in the Object Explorer and select Refresh, and our newly created table will show in the list.

Once this is done we can Right Click on our newly created capitals table and select View/Edit Data to view our table. Notice that when we tell pgAdmin to do something, the SQL Code to complete the process shows up un the "Query Window".

Lastly, we want to Import our data from the "capitals.csv" file we were given for this assignment.

  1. Right click on our capitals table and select Import/Export Data.
  2. On the General tab, under "Filename", locate the file we are importing.
  3. Because our csv file contains a "Header" row, we need to go to the Options tab and check the "Header" box.
  4. *NOTE: it's important to note that our column names from the csv file must match exactly the column names we used when build the table. You can check the names on the Columns tab.

And now when we right click on our table, and select "View/Edit" we should see the table populated with the data from our csv file.

We duplicated the process to create a "Flags" table populated with data from the "flags.csv" file. Here is the Query Code:

  • CREATE TABLE flags (
    • id SERIAL PRIMARY KEY,
    • name text,
    • flag text
  • )

We now have two tables (capitals and flags) populated with the data from our two csv files.
