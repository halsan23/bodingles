Type Pseudo Class Selector

Return to CSS Tags Index
    Type pseudo-classes:
  • Style elements by their tag type
  • Different from state or position-based pseudo-classes
  • Targets elements based on their nature or tag
    Examples:
  • :is()
    • Matches any of given selectors
    • :is(div, p) targets both < div> & < p> tags
  • :not()
    • Matches elements not matching given selectors
    • :not(.spoc) targets all except the .spoc class
  • :where()
    • like :is() but doesn't affect specificity
    • :where(div, p) for both, no specificity change
    Analogy
  • Type pseudo-classes as magical glasses for toys
  • Find red toys pr exclude the blue toys
  • Pseudo-classes help find specific items
    When to Use
  • :is() - Style multiple elements without repitition
  • :not() - Exclude certain elements from a style
  • :where() - Match like :is() without specificity change
    Best Practices
  • May not work on older browsers
  • Enhance maintainability and conciseness with pseudo-classes
  • Consider Specificity impact of :is() and :not()
    Take Note
  • :is() - Reduce style redundancy, manageability
  • :not() - Versatile but use judiciously to avoid complexity
  • :where() - Match without specificity impact, more styling control
