Including Bootstrap and Containers

Installing Bootstrap

You can utilize bootstrap by three methods:

  • Use a "stylesheet" link in your < head> section of your webpage.
    • called the CDN method
    • If you install via this method, put the stylesheet link before any of your local stylesheets for local, custom, stylesheet control.
    • Link Here
  • You can download bootstrap and incorporate it into your web files.
    • Link Here
  • You use npm to install Bootstrap into your VsCode environment
    • *note: you must have Node.js installed prior to installing via npm
    • Link Here

Some Bootstrap elements require JavaScript be utilized for the elements to work properly. The JavaScript link can be found under the CDN < head> link, and must be placed at the bottom of the HTML page as the last line of code in the < body> tag.

Containers

Containers are the most basic layout element in Bootstrap and are required when using our default grid system. Containers are used to contain, pad, and (sometimes) center the content within them. While containers can be nested, most layouts do not require a nested container.

Bootstrap comes with three different containers:

  • .container, which sets a max-width at each responsive breakpoint
  • .container-fluid, which is width: 100% at all breakpoints
  • .container-{breakpoint}, which is width: 100% until the specified breakpoint
