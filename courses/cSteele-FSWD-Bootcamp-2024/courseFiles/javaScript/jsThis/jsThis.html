JavaScript Events & This
Events & this

W3Schools this
MDN this

this is a keyword that essentially lets us process several different types of elements with one universal section of code. In JavaScript, the this keyword refers to an object.

The this keyword refers to different objects depending on how it is used:

  • In an object method, this refers to the object.
  • Alone, this refers to the global object.
  • In a function, this refers to the global object.
  • In a function, in strict mode, this is undefined.
  • In an event, this refers to the element that received the event.
Methods like call(), apply(), and bind() can refer this to any object.

*Note: this is NOT a variable. It is a keyword. You cannot change the value of this.

The this keyword refers to the context where a piece of code, such as a function's body, is supposed to run. Most typically, it is used in object methods, where this refers to the object that the method is attached to, thus allowing the same method to be reused on different objects.

Arrow functions differ in their handling of this: they inherit this from the parent scope at the time they are defined. This behavior makes arrow functions particularly useful for callbacks and preserving context. However, arrow functions do not have their own this binding. Therefore, their this value cannot be set by call(), apply() or bind() methods, nor does it point to the current object in object methods.



In the following example, we have a section set up with several boxes. These boxes are just simple HTML buttons with some CSS styling,

  • <button>CLICK</button>

Now for the JavaScript.

1). First we need to define a function to create the random foreground and background colors:

  • const makeRandColor = () => {
    • // picks random r/g/b colors and assigns them to fc (foreground color)
    • let r = Math.floor(Math.random() * 255);
    • let g = Math.floor(Math.random() * 255);
    • let b = Math.floor(Math.random() * 255);
    • const fc = `rgb(${r}, ${g}, ${b})`;
    •  
    • // picks random r/g/b colors and assigns them to bc (background color)
    • r = Math.floor(Math.random() * 255);
    • g = Math.floor(Math.random() * 255);
    • b = Math.floor(Math.random() * 255);
    • const bc = `rgb(${r}, ${g}, ${b})`;
    •  
    • // returns the random fc and bc colors
    • return [fc, bc];
  • }

2). Next we need to define the button objects:

  • const buttons = document.querySelectorAll('button');
    • so buttons becomes an object that represents all of the buttons.

3). Next we need to create a for loop so that we can assign a click event to each individual button.

  • for (let button of buttons) {
    • button.addEventListener('click', colorize)
  • }
    • since we know that the addEventListener() syntax is:
      • variable.addEventListener('typeOfEvent', callbackFunction)
        • colorize becomes our callbackFunction for our addEventListener

4). So next we need to define the callbackFunction colorize

  • function colorize() {
    • const [fc, bc] = makeRandColor();
    • this.style.color = fc;
    • this.style.backgroundColor = bc;
  • }
  • So colorize() will run the imbedded function makeRandColor() which actually creates the random foreground (fc) and background (bc) colors.
  • Instead of applying these colors to the button variable, we are applying them to the keyword this.
  • Because in this instance, this is called in an Object Method, it will refer TO the object button.
